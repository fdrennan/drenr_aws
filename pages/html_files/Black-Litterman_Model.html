# Create D Matrix
D_Mat <- 2*C_Mat

# Create A Matrix
Proportion_Vector <- rep(1, length(Historical_Returns))
Diag_Matrix <- diag(1, length(Historical_Returns))
A_Mat <- rbind(Historical_Returns, -Historical_Returns, 
               Proportion_Vector, -Proportion_Vector, Diag_Matrix)

# Create D_Vector
D_Vector <- rep(0, length(Historical_Returns))

# Create starting point for estimated returns (cycle through for many returns)
ret <- min(Historical_Returns)

# How many points in between we would like from min return to max return
Number_Of_Points <- 20

# Create vectors to create stacked area proportion chart
Chosen_Return <- vector()
Percentage <- vector()

# While loop to run program and get back many returns
while(ret <= max(Historical_Returns)) {
  
  # Create B_Vector
  B_Vec <- c(ret, -ret, 1, -1, rep(0, length(Historical_Returns)))
  
  # Run QP
  sol <- solve.QP(D_Mat, D_Vector, t(A_Mat), B_Vec)
  
  # Put Standard Deviation and Expected Return into a data frame for plotting
  Portfolio_Standard_Deviation <- sqrt(sol$value)
  Portfolio_Expected_Return <- sum(Historical_Returns*sol$solution)
  Efficient_Frontier_One <- rbind(c(Portfolio_Expected_Return, 
                                Portfolio_Standard_Deviation), 
                              Efficient_Frontier_One)
  colnames(Efficient_Frontier_One) <- c("Estimated_Return", "Std_Deviation")
  
  # Creating Stacked Area Chart data
  Chosen_Return <- append(rep(ret, length(sol$solution)), Chosen_Return)
  Percentage <- append(sol$solution, Percentage)
  
  # Add to index variable
  ret <- ret + (max(Historical_Returns) - min(Historical_Returns))/Number_Of_Points
}

# Sector variable for stacked area chart
Sector <- rep(Names, (Number_Of_Points)+1)

The Black-Litterman model allows us to express views either in relative or absolute terms. These views are set in the P matrix, where we write a series of relationships to describe a particular view. For example, the first view is an absolute view: \(x_6 = .061\) says a US Small Value will have a 6.1% premium. The second view is relative: \(-x_7 + x_8 = .03\) requires International Emerging Equity to be larger than International Devoped Equity by 3%, though the specific numbers can vary as long as the difference remains the same. The resulting solution \(\hat{m}\) appears to be a weighted average of our view and the historical returns.

If \(\tau\) is very small, the views we express in the P matrix will be weighted less into the model. If one chooses \(\tau\) of .001, the efficient frontier will nearly resemble the efficient frontier for the initial portfolio. The smaller tau is, the more weight is given to the historical returns. This is because when we multiply a small scaler by a matrix, then solve for the inverse, the resulting matrix is large. This large matrix is then multiplied by the historical returns. An example of how a small scaler creates a larger matrix when the inverse is taken is shown below.

sum(solve(.001*C_Mat))
## [1] 1179687
sum(solve(5*C_Mat))
## [1] 235.9373

It is also worth noting that the efficient frontier for \(\tau\) = 1 is nearly the same as \(\tau\) = 100 and so appears that decent range for \(\tau\) might be somewhere between .1 and .5. The greater \(\tau\) is, the further to the northwest the efficient frontier moves because we have stated that we believe US Small Value with have a greater return than the historical returns tell us. If we decided to state that \(x_6 = -.01\) then the efficient frontier would decrease given that returns are expected to be lower.
